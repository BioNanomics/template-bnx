## Template Usage Notes

When using this template for a new project:

1. Update the `[Unreleased]` section with your initial changes
2. Replace the template version history with your project's actual releases
3. Follow the format for each release:
   - Use semantic versioning (MAJOR.MINOR.PATCH)
   - Include the release date
   - Categorize changes as Added/Changed/Deprecated/Removed/Fixed/Security
4. Link to compare views and release tags when available

### Example Entry Format:

```markdown
## [1.2.3] - 2024-MM-DD

### Added
- New feature description

### Changed
- Modified feature description

### Fixed
- Bug fix description
```
